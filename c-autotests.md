# Как подготовить исходный текст программы

Вот пример задания:

![Задание](/assets/images/c-autotests-example.png)

Оно состоит из следующих частей:

* Описание -- что нужно сделать
* Входные параметры -- что и в каком порядке будет вводиться в программу
* Выходные параметры -- какого формата должен быть вывод

С описанием понятно, давай рассмотрим "входные параметры".

### Входные параметры

В задании указан размер массива "20" -- это значит что массив может быть от 1 до 20 элементов.

Система автотестов случайным образом определяет размер массива в заданном промежутке и эмулирует его ввод с клавиатуры. Например: 5

Затем, система генерирует случайные значения типа из столбца "Тип", которые расположены в промежутке, указанном в столбце "Допустимые значения".

То есть, чтобы прочитать входные параметры надо:

* Считать размер массива
* Считать элементы массива

Этот код будет выглядеть так:

```c
int n;
scanf("%d", &n);
if(n < 1 || n > 20)
	return 0;

int arr[20];
for(int i = 0; i < n; i++)
	scanf("%d", &arr[i]);
```

Ну всё, можно обрабатывать значения в соответствии с описанием задачи.

### Выходные параметры

В задании сказано вывести 1 или 0 -- вроде понятно.

Однако, есть одна особенность -- **перед выводом результата добавь перенос на новую строку**:

```c
int n;
scanf("%d", &n);
if(n < 1 || n > 20)
	return 0;

int arr[20];
for(int i = 0; i < n; i++)
	scanf("%d", &arr[i]);

char res = 1;
// Вычисления...

// Перенос на новую строку, чтобы отделить весь прошлый вывод программы
printf("\n");
if(res)
	printf("1");
else
	printf("0");
```

Это обусловлено тем, что автотесты проверяют только последнюю строку в выводе твоей программы.

# Особенности входных параметров

Здесь, я приведу различные виды входных параметров, которые могут встретиться в заданиях.

### Многомерные массивы

Массивы могут быть не только одномерными, но и двух и трёхмерными.

В столбце "Размер массива" они обозначаются так: 5x10, 5x10x4 и так далее...

Если числа в размере разные, то их надо считывать по отдельности:

* 5x10 -- надо считать первую размерность (количество строк), потом вторую (количество столбцов)
* 9x9 -- надо считать только первую размерность, так как обе размерности одинаковые (квадратная матрица)

### Несколько входных параметров

На вход могут идти не только массивы, но и числа:

![Несколько параметров](/assets/images/c-autotests-example-multiple.png)

В таком случае, считывай их с том же порядке, в каком они указаны. Если рассмотреть указанный пример:

* Считай массив так, как указано выше
* Считай целочисленное n
* Считай целочисленное m

# Автотесты находятся в бета-версии

Автотесты ещё только встают на ноги, так что отнесись с пониманием.

Автотесты работают по принципу сравнения результатов двух программ: эталонной и твоей. Так как эталонные программы ещё не обкатаны -- они могут содержать ошибки.

Поэтому, если автотесты не хотят пропускать твоё решение, а ты уверен что оно правильное, то пиши мне ([контакты](/about-ru)) сразу -- я проверю и починю автотесты.
